Frequently Asked Questions

What is the difference between var, let, and const?

var is function-scoped, can be re-declared, and is hoisted. let is block-scoped, cannot be re-declared, and is not hoisted. const is block-scoped, cannot be re-declared or re-assigned, but its properties (if an object or array) can be modified.

What is the difference between map(), forEach(), and filter()?

map() creates a new array by applying a function to each element. forEach() executes a function for each element but does not return a value. filter() creates a new array with elements that pass a test.

What are arrow functions, and how are they different from regular functions?

Arrow functions are a shorter way to write functions. They do not have their own this (they inherit it from the surrounding scope), cannot be used as constructors, and do not have an arguments object.

How do JavaScript Promises work?

Promises handle asynchronous operations. They have three states: pending (initial state), fulfilled (operation succeeded), and rejected (operation failed). You use .then() to handle success and .catch() to handle errors.

How do closures work in JavaScript?

Closures allow a function to "remember" the environment in which it was created. They are useful for data encapsulation, creating private variables, and implementing callbacks.
